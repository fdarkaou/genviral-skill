# genviral Partner API Skill

> **TL;DR for agents:** This skill wraps genviral's Partner API into 32 bash commands. Core workflow: `generate` (create slideshow from prompt) > `render` (produce images) > `review` (check quality) > `create-post` (publish). Auth via `GENVIRAL_API_KEY` env var. Config in `config.yaml`. Product context in `context/`. Hook library in `hooks/`. Track results in `performance/`. The skill self-improves: post > track > analyze > adapt strategy > post better content.

Complete automation for genviral's Partner API. Create video posts, AI-generated slideshows, manage templates and image packs, and schedule content across any platform genviral supports (TikTok, Instagram, etc.).

This skill provides a full CLI wrapper around the Partner API with commands for every endpoint, plus higher-level workflows for content creation, performance tracking, and strategy optimization.

## What This Skill Does

- **Multi-Platform Posting:** Create posts for TikTok, Instagram, or any connected account (video OR slideshow, multiple accounts per post)
- **File Management:** Upload videos/images to genviral's CDN with presigned URL flow
- **AI Slideshow Generation:** Generate photo carousels from prompts, render them to images
- **Template System:** Create reusable slideshow templates, convert winning slideshows to templates
- **Pack Management:** Manage image packs (backgrounds for slideshows)
- **Content Pipeline:** Full automation from prompt to posted draft
- **Performance Tracking:** Log posts, track metrics, learn from results
- **Hook Library:** Maintain and evolve a library of proven content hooks

## How It Works

The core workflow is:

1. **Generate or upload media** (slideshow from prompt, or upload your own video/images)
2. **Create a post** targeting one or more accounts
3. **Schedule or publish** (immediately or at a specific time)
4. **Track performance** (views, engagement at 24h, 48h, 7d intervals)
5. **Learn and optimize** (promote winning hooks, retire underperformers)

The skill handles the full automation. For TikTok slideshow posts, it can optionally save as drafts so you add trending audio before publishing (music selection requires human judgment for best results).

### What Each File Does

**config.yaml** holds all technical configuration: base URL, default pack ID, slide count, aspect ratio, style preset, account IDs, posting schedule, hashtag pools, and performance thresholds. Secrets (API key) are loaded from environment variables.

**context/product.md** describes the product being promoted. What it does, who it serves, what problem it solves, key features, value proposition.

**context/brand-voice.md** defines tone and style. Casual or professional? Funny or serious? What words to use/avoid, examples of ideal voice.

**context/niche-research.md** is auto-generated by the agent. Contains research on what works on each platform: competitor accounts, trending formats, hashtag analysis.

**hooks/library.json** stores all hook instances with metadata: hook text, formula used, times used, performance data. The agent draws from this library and evolves it based on results.

**hooks/formulas.md** documents underlying hook patterns (person+conflict, educational, before/after, etc.) with placeholders for generating new hooks.

**content/scratchpad.md** is the agent's working memory for content planning. Current batch ideas, draft hook lines, notes on what to try next.

**content/calendar.json** is the scheduled content plan. Each entry has a date, time slot, hook ID, status (planned/generated/posted), and related IDs.

**performance/log.json** tracks every post with timestamps, hook used, slideshow ID, post ID, and metrics at each checkpoint (24h, 48h, 7d).

**performance/insights.md** contains distilled learnings: which hook formulas perform best, which visual styles get engagement, optimal posting times.

**performance/weekly-review.md** is the template for weekly review process. The agent fills this in every 7 days to evaluate performance and plan the next week.

## Script Reference

All commands use the wrapper script:

```bash
/path/to/genviral/scripts/genviral.sh <command> [options]
```

The script requires `GENVIRAL_API_KEY` as an environment variable. It loads defaults from `config.yaml`.

### Account & File Commands

#### accounts
List connected BYO and hosted accounts in your scope. Use this to discover account IDs for posting.

```bash
genviral.sh accounts
genviral.sh accounts --json
```

Returns:
- Account ID (use in --accounts for create-post)
- Platform (tiktok, instagram, etc.)
- Type (byo or hosted)
- Username, display name, status

#### upload
Upload a file to genviral's CDN using the presigned URL flow. Returns a CDN URL you can use in posts.

```bash
genviral.sh upload --file video.mp4 --content-type video/mp4
genviral.sh upload --file slide1.jpg --content-type image/jpeg --filename "slide1.jpg"
```

Supported content types:
- Videos: `video/mp4`, `video/quicktime`, `video/x-msvideo`, `video/webm`, `video/x-m4v`
- Images: `image/jpeg`, `image/png`, `image/gif`, `image/webp`, `image/heic`, `image/heif`

Returns the CDN URL (use in create-post).

#### list-files
List files uploaded via the Partner API.

```bash
genviral.sh list-files
genviral.sh list-files --type video --limit 20 --offset 0
genviral.sh list-files --context all  # include all contexts
genviral.sh list-files --json
```

### Post Commands

#### create-post
Create a post (video OR slideshow) targeting one or more accounts. This is the core posting command.

**Video post:**

```bash
genviral.sh create-post \
  --caption "Your caption with #hashtags" \
  --media-type video \
  --media-url "https://cdn.genviral.com/your-video.mp4" \
  --accounts "account_id_1,account_id_2" \
  --scheduled-at "2025-03-01T15:00:00Z"
```

**Slideshow post:**

```bash
genviral.sh create-post \
  --caption "Your caption" \
  --media-type slideshow \
  --media-urls "url1,url2,url3,url4,url5,url6" \
  --accounts "account_id_1" \
  --music-url "https://www.tiktok.com/@user/video/1234567890"
```

**TikTok-specific settings** (only when ALL accounts are TikTok BYO):

```bash
genviral.sh create-post \
  --caption "Caption" \
  --media-type video \
  --media-url "https://..." \
  --accounts "tiktok_account_id" \
  --tiktok-title "Optional title" \
  --tiktok-description "Optional description" \
  --tiktok-post-mode "MEDIA_UPLOAD" \
  --tiktok-privacy "SELF_ONLY" \
  --tiktok-disable-comment \
  --tiktok-disable-duet \
  --tiktok-disable-stitch
```

TikTok `post_mode` options:
- `DIRECT_POST` - publish immediately (default)
- `MEDIA_UPLOAD` - save to TikTok drafts inbox (requires user to add music and publish)

TikTok `privacy_level` options:
- `PUBLIC_TO_EVERYONE` (default)
- `MUTUAL_FOLLOW_FRIENDS`
- `FOLLOWER_OF_CREATOR`
- `SELF_ONLY` (draft mode)

**Scheduling:**

- Omit `--scheduled-at` or set it within 30 seconds of now: post is queued for immediate publish (status: `pending`)
- Provide future ISO timestamp: post is scheduled (status: `scheduled`)

**Multi-account posting:**

You can target up to 10 accounts per post. Mix TikTok, Instagram, etc. Music is only supported when ALL accounts support it (currently TikTok only). TikTok-specific settings only work when ALL accounts are TikTok BYO.

Returns:
- Post ID
- Status (pending, scheduled)
- Warnings (if any, e.g. video duration metadata missing)

#### update-post
Update an existing post (only editable if status is draft, pending, scheduled, retry, or failed).

```bash
genviral.sh update-post \
  --id POST_ID \
  --caption "Updated caption" \
  --media-type video \
  --media-url "https://new-video.mp4" \
  --accounts "new_account_id_1,new_account_id_2" \
  --scheduled-at "2025-03-15T18:00:00Z"
```

To remove music: `--music-url null`

#### retry-posts
Retry failed or partial posts.

```bash
genviral.sh retry-posts --post-ids "post_id_1,post_id_2"
genviral.sh retry-posts --post-ids "post_id_1" --account-ids "account_id_1"
```

Returns:
- Number of posts retried
- IDs of successful retries
- Errors for any failed retries

#### list-posts
List posts with optional filters.

```bash
genviral.sh list-posts
genviral.sh list-posts --status scheduled --limit 20
genviral.sh list-posts --since "2025-02-01T00:00:00Z" --until "2025-02-28T23:59:59Z"
genviral.sh list-posts --json
```

Status filters: `draft`, `pending`, `scheduled`, `posted`, `failed`, `partial`, `retry`

Returns:
- Summary (total, pending, published, counts by status)
- Posts array

#### get-post
Get details for a specific post.

```bash
genviral.sh get-post --id POST_ID
```

Returns full post object with per-account states.

#### delete-posts
Bulk delete posts by IDs.

```bash
genviral.sh delete-posts --ids "post_id_1,post_id_2,post_id_3"
```

### Slideshow Commands

#### generate
Generate a slideshow from a prompt using AI.

```bash
genviral.sh generate \
  --prompt "Your hook and content prompt" \
  --pack-id PACK_ID \
  --slides 6 \
  --type educational \
  --aspect-ratio 9:16 \
  --style tiktok \
  --language en \
  --font-size 48 \
  --text-width 80
```

All options except `--prompt` fall back to config.yaml defaults.

Options:
- `--slides` - number of slides (1-10, default: 6)
- `--type` - `educational` or `personal`
- `--aspect-ratio` - `9:16` (vertical), `4:5` (IG feed), `1:1` (square)
- `--style` - `tiktok` (white text + shadow), `inverted`, `shadow`, `white`, `black`
- `--language` - language code (e.g., `en`, `es`, `fr`)
- `--font-size` - custom font size (pixels)
- `--text-width` - custom text width (percentage)

Returns:
- Slideshow ID
- Full slideshow object with slides and text

#### render
Render a slideshow to images via Remotion. Takes about 2-5 seconds per slide.

```bash
genviral.sh render --id SLIDESHOW_ID
```

Returns:
- Updated slideshow with rendered image URLs
- Status: `rendered`

#### review
Get full slideshow details for review. Shows slide text, status, rendered URLs.

```bash
genviral.sh review --id SLIDESHOW_ID
genviral.sh review --id SLIDESHOW_ID --json
```

Without `--json`, prints a human-readable summary to stderr and JSON to stdout.

#### update
Update slide text or title. Re-render after updating.

```bash
genviral.sh update --id SLIDESHOW_ID \
  --slides '[{"index": 0, "text": "New text for slide 1"}]'

genviral.sh update --id SLIDESHOW_ID --title "New Title"
```

#### regenerate-slide
Regenerate AI text for a single slide (0-indexed).

```bash
genviral.sh regenerate-slide --id SLIDESHOW_ID --index 2
```

#### duplicate
Clone an existing slideshow as a new draft.

```bash
genviral.sh duplicate --id SLIDESHOW_ID
```

Returns the new slideshow ID.

#### delete
Delete a slideshow.

```bash
genviral.sh delete --id SLIDESHOW_ID
```

#### list-slideshows
List slideshows, optionally filtered by status.

```bash
genviral.sh list-slideshows
genviral.sh list-slideshows --status rendered
genviral.sh list-slideshows --json
```

## Content Creation Pipeline

This is the recommended workflow for producing posts. The agent should follow these steps in order.

### For Slideshow Posts

1. **Hook Selection:** Read `hooks/library.json` and pick a hook. Rotate through categories. Prioritize hooks that have not been used recently.

2. **Prompt Assembly:** Use the selected hook to build a full slideshow prompt. Reference `prompts/slideshow.md` for templates. Read `context/product.md` and `context/brand-voice.md`.

3. **Generate Slideshow:** Run `generate` with the assembled prompt.

4. **Review Slide Text:** Check each slide for clarity, readability, flow. Update or regenerate weak slides.

5. **Render:** Run `render` to generate final images.

6. **Visual Review:** Check rendered images for text readability, contrast, no overflow.

7. **Post:** Use `create-post` with media-type slideshow, or use legacy `post-draft` for TikTok drafts.

8. **Notify:** Send notification to user with caption, draft link/ID, reminder to add music (if TikTok draft).

9. **Log:** Record the post in `performance/log.json` with date, hook, slideshow ID, post ID, caption, hashtags, status.

10. **Track Performance:** After 24h, 48h, 7d, check metrics and update log.

### For Video Posts

1. **Source Video:** Upload via `upload` command or reference existing CDN URL.

2. **Write Caption:** Follow brand voice, include relevant hashtags, keep it concise.

3. **Create Post:** Run `create-post` with media-type video, target accounts, optional scheduling.

4. **Log:** Record in `performance/log.json`.

5. **Track Performance:** Check metrics at intervals, update log.

## Hook Formulas

The skill uses five core hook formulas. These are patterns, not specific hooks. The agent generates specific hooks by combining a formula with the product context.

See `hooks/formulas.md` for full documentation. Summary:

### 1. Person + Conflict (Primary Driver)

`[Person] said [doubt or objection about PRODUCT]. So I showed them [what the product does]. [Reaction/Result].`

Highest-performing formula for discovery content. Creates instant curiosity through interpersonal tension.

### 2. Educational / Tips

`[Number] [things/mistakes/hacks] about [TOPIC] that [surprising claim]`

Positions the brand as helpful and knowledgeable. Works well for saves and follows.

### 3. Before / After

`[Metric/situation] before [PRODUCT] vs after. [Specific improvement].`

Transformation content is inherently shareable. Specificity (real numbers, real timelines) builds credibility.

### 4. Relatable Pain Point

`That moment when [universal frustration related to PRODUCT's problem space]...`

Drives comments and shares because people tag friends who relate.

### 5. POV / Lifestyle

`POV: You [aspirational or relatable scenario that PRODUCT enables]`

Places the viewer inside the experience. Works best when it triggers either aspiration or recognition.

## Platform Best Practices

### TikTok

**Slide Count:** 6 slides is the sweet spot. Quick tips: 5 slides. Long form: max 7 slides.

**Aspect Ratio:** Always 9:16 (vertical fullscreen).

**Text Readability:** One idea per slide. 8-16 words max. Use "tiktok" style preset (white text with drop shadow). Avoid text in bottom 20% (UI overlap).

**Narrative Structure (6-slide arc):**
1. Hook (stop the scroll)
2. Problem detail (make them feel the pain)
3. Shift (introduce the solution)
4. Feature/proof 1
5. Feature/proof 2 or result
6. CTA (soft, link in bio)

**Caption Formula:** Short. Slideshow is the content, caption is reinforcement.

```
[Hook restated in one sentence]. [Outcome or value promise].

[3-5 niche hashtags]
```

**Hashtag Strategy:**
- 1-2 "always on" branded/niche hashtags
- 3-4 rotating hashtags from pool
- Mix broad discovery tags (#fyp, #lifehack) with niche-specific tags
- Track which combinations drive reach
- Update pool monthly

**Posting Schedule:**
- 2-3 posts per day during growth phase
- Space posts 3-4 hours apart
- Test morning (9-11am), afternoon (1-3pm), evening (6-9pm) in target timezone
- US-focused: peak 9-11am ET and 5-8pm ET

**Music (Human Step):**
- Always add trending sound before publishing (for draft posts)
- Check TikTok's "popular sounds" in editor
- Match energy: upbeat for transformation, chill for tips
- This can 2-3x reach

### Instagram

**Slide Count:** 5-10 slides for carousel posts. Single slide for Reels.

**Aspect Ratio:**
- Reels: 9:16
- Feed posts: 4:5 or 1:1
- Stories: 9:16

**Text Readability:** Similar to TikTok but slightly more text-tolerant. Keep under 20 words per slide.

**Caption:** Instagram allows longer captions. Use first 125 characters as hook (appears in feed without "...more").

**Hashtags:** 5-10 relevant hashtags. Place at end of caption or in first comment.

**Posting Schedule:**
- 1-2 posts per day
- Best times: 11am-1pm, 7-9pm in target timezone

**Cross-Posting:** You can use the same slideshow for TikTok and Instagram. Adjust aspect ratio and caption if needed.

### Multi-Platform Strategy

For maximum reach, create once and distribute everywhere:

1. Generate slideshow with 9:16 aspect ratio
2. Render once
3. Post to TikTok + Instagram Reels with same slideshow, platform-specific captions
4. For IG feed, re-generate with 4:5 aspect ratio
5. Track performance separately per platform
6. Learn which hook formulas work best on each platform

## Self-Improvement Loop

The skill is designed to get better over time.

### The Feedback Cycle

```
Generate hooks --> Create posts --> Track performance --> Analyze results --> Update strategy --> Generate better hooks
```

Each post produces data. After enough data (7 days or 15+ posts), run a review cycle.

### Weekly Review Process

Every 7 days, the agent should:

1. **Pull metrics** for all posts from the past week
2. **Rank by performance** (views, saves, shares at 24h and 7d)
3. **Identify top 3 posts.** What hook formula? What made them work?
4. **Identify bottom 3 posts.** What went wrong?
5. **Update insights.** Write findings to `performance/insights.md`
6. **Refresh hooks.** Generate 5-10 new hooks modeled after top performers. Retire hooks that failed >2 times.
7. **Plan next week.** Update `content/calendar.json` weighted 70% toward proven formulas, 30% experiments.
8. **Update niche research.** Check if trends, competitor strategies, or hashtags have shifted.

Use `performance/weekly-review.md` as the template.

### Hook Lifecycle

Each hook goes through stages:

- **New:** Just created, never used. Priority for testing.
- **Active:** Used 1-2 times, performance data available.
- **Proven:** Consistently strong performance. Use as model for variants.
- **Retired:** Used max times or consistently underperformed. Archived for reference.

Maintain a healthy mix: 30% new (experiments), 50% active (iterating), 20% proven (reliable performers).

### Strategy Evolution

Over weeks/months, the agent accumulates enough data to identify meta-patterns:

- Which hook formula consistently wins for this product/niche?
- What time of day drives most views?
- Which visual pack gets best engagement?
- What CTA style drives most profile visits?
- Seasonal or trend-based patterns?

Distill into `performance/insights.md` and use to refine the entire approach.

## Quality Checklist

Before posting any content, verify:

- [ ] Hook line sounds human and specific (not generic or corporate)
- [ ] Slide 1 is understandable in under 1 second
- [ ] Each slide has one clear idea, max 16 words
- [ ] Narrative flows logically from hook to CTA
- [ ] No AI-sounding phrases ("harness the power of", "revolutionize your")
- [ ] Product mention feels natural, not forced
- [ ] CTA is short and clear
- [ ] Caption restates the hook, includes relevant hashtags
- [ ] Rendered images are readable and visually clean

## Troubleshooting

**"GENVIRAL_API_KEY is required"**
Export the environment variable: `export GENVIRAL_API_KEY="your_public_id.your_secret"`
Or add to `~/.config/env/global.env`.

**"No rendered image URLs found"**
The slideshow has not been rendered yet. Run `render` first.

**"account_id is required"**
Set default account IDs in `config.yaml` under `posting.default_account_ids`, or pass `--account-ids` to the command.

**API returns 401 or 403**
Verify API key format (should be `public_id.secret`). Check that the partner account has correct permissions in genviral.

**Render takes too long or times out**
Each slide takes 2-5 seconds. For 6 slides, expect up to 30 seconds. Increase HTTP timeout in config if needed.

**Slides have text overflow or readability issues**
Shorten text (under 12 words per slide), adjust `font_size` or `text_width` in config, or try a different style preset.

**Music blocked when posting to Instagram**
Music URLs are only supported for TikTok. Remove `--music-url` when targeting Instagram accounts.

**TikTok settings rejected**
TikTok-specific settings only work when ALL selected accounts are TikTok BYO accounts. Remove TikTok settings or post to TikTok-only.

## Notes

- **Multi-platform support:** This skill works with any platform genviral supports (TikTok, Instagram, etc.). Account discovery and posting logic are platform-agnostic.
- **Content types:** Supports both video posts and slideshow (photo carousel) posts. Slideshows are AI-generated text-on-image carousels. Videos are uploaded separately.
- **Hosted vs BYO accounts:** Works with both hosted (managed virtual accounts) and BYO (bring your own connected accounts). Mix and match in the same post.
- **Scheduling:** Posts can be scheduled for future publish or queued for immediate posting. Scheduling uses ISO 8601 timestamps.
- **Draft mode:** For TikTok, use `post_mode: MEDIA_UPLOAD` to save to drafts inbox. User adds music and publishes manually. This is intentional for best audio selection.
- **Template system:** Convert winning slideshows to templates for faster iteration. Templates preserve structure and visual style.
- **Performance tracking:** The skill includes optional performance tracking and strategy evolution. This is separate from posting and can be used or ignored.
- **Hook library:** The hook system is optional but highly recommended for consistent content quality and improvement over time.
- **Never use em-dashes** in any generated content. Use periods, commas, or restructure sentences instead.
